Test Case Generation
	
	The generation of test cases for API is extremely hard to do in an automatic capacity. API documentation is extremely large but it can also be rather abstract. It is not easy to teach a program the relation of different API calls from the documentation alone. For this reason we have kept test case generation to a semi manual process. A test case for our program consists of one or more API calls that should satisfy a metamorphic relationship and test data for these calls. To generate a test case the API documentatio is analysed by one of our members. The purpose of this analysis is to identify API calls that are linked with metamorphic relationships that we have described (Equiality, Equivalence, Subset, Disjoint, Complete and Difference). Once a relation has been found the calls are broken down into a format that we can then test. This usually consists of a base URL and test data that is sent to the backend for testing. The backend then returns a result pertaining to the validity of the test cases metamorphic relationship.
